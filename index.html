HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • Declarations are NOT case sensitive
  • HTML attribute defines additional info about element
  • Style defines css, base defines base URL, link connects external resource, script embeds client-side scipts
  • Meta defines metadata such as viewport, author, charset, description, keywords
  • Element refers to entire element from opening tag to closing tag
  • Tag refers to only what's inside angle brackets
  • The rel attribute specifies the relationship between the current document and the linked resource
  • The href attribute specifies the location of the external resource

CSS

  • Inline CSS applies to a specific HTML tag and is less flexible
  • Internal CSS contains rules for the webpage in the head section of the HTML file
  • External CSS keeps all CSS rules in a separate file which makes design changes easier
  • The period preceding class name designates this as a class selector
  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git clone ssh-name: clones ssh to directory
  • git add -a: adds all changes in working directory to staging directory
  • git push branch-name: push local changes
  • git pull origin main: receive branch modification into local environment to make sure it's up-to-date
  • git commit -m: describe intention of commit

JavaScript

  • Variables are containers that store values and you declare them with the "let" keyword followed by the name of the variable
  • String - enclosed with single or double quotation marks
  • Numbers - don't have quotes around them
  • Boolean - the words true and false are special keywords that don't need quote marks
  • Array - this is a structure that allows you to store multiple values in a single reference
  • Object - this can be anything; everything in JavaScript is an object and can be stored in a variable
  • Comments - // ___ for no line breaks else /*,*/
  • An operator is a mathematical symbol that produces a result based on two values (or variables)
  • Conditionals are code structures used to test if an expression returns true or not
  • Functions are a way of packaging functionality that you wish to reuse
  • Arguments go inside the parentheses, separated by commas if there is more than one argument
  • Conceptually, undefined indicates the absence of a value, while null indicates the absence of an object
  • The const keyword allows us to create variables that can't be reassigned a value
  • Note that whereas the style.css file is linked within the head element, the script.js file is linked within the body element
